## recursive-index/gmsplatform/onboarding
The GMS Platform Onboarding directory contains a BoQ-based automation service designed to streamline the infrastructure setup for GMS Core modules and feature launches. This service provides a centralized RPC interface, defined in the proto subdirectory, that orchestrates complex workflows across multiple Google internal systems including repository for version control, Rapid for release management, L-Space for artifact storage, and Gantry for project tracking and ACL management. By automating tasks such as mobilespec setup, feature flag configuration, and metric integration (e.g., Bottomline and Install Count), the service reduces the manual effort required from client teams during the module onboarding and rollout phases.
Architecturally, the directory is organized into specialized components that collaborate to provide this automation. The service subdirectory contains the core business logic implemented as BoQ Actions, which leverage a suite of shared utilities in common for interacting with repository, executing CodeMaker schematics for template-based code generation, and formatting files via fmtserver.
The client and keyregistryclient subdirectories provide the necessary communication layers for external consumers and internal dependencies, while server functions as the composite node for assembling the service into a deployable production unit. This structure facilitates a modular approach to infrastructure orchestration, supported by an asynchronous promisegraph framework that manages the dependencies between various infrastructure mutations.

# Root Map - Epoch 1
## Summary
The recursive-index/gmsplatform/onboarding* directory encapsulates a BoQ-based automation service designed to orchestrate and streamline the infrastructure lifecycle for GMS
Core modules and feature launches. Operating as a centralized hub, it leverages a unified RPC interfacedefined within the proto/ subdirectoryto automate complex, multi-system workflows across Google's internal ecosystem, including repository for version control, Rapid for release management, L-Space for artifact storage, and Gantry for project and Access Control List (ACL) management.
By programmatically executing tasks such as Mobilespec configuration, BCID policy creation, and telemetry onboarding for critical metrics like Bottomline and Client Tracing, the service ensures consistent, error-free environment provisioning for various client teams. Architecturally. the codebase is modularized to support scalability and maintainability: the service/ directory implements core business logic through BoQ Actions, while common provides essential abstraction layers for repository operations, CodeMaker-driven file generation, and TechFile interactions. Specialized interfaces for service consumption and signing key metadata retrieval are housed in 'client and keyregistryclient", respectively, all of which converge into a deployable unit within the server directory.
A defining feature of this service is its use of the "promisegraph" framework, which enables the parallelization of multi-system mutations and the automation of traditionally interactive command-line tools such as "mppli". Consequently, this directory functions as a stateful automation agent that replaces manual infrastructure management with a programmatic, reliable, and integrated pipeline. This approach significantly reduces the operational overhead associated with GMS Platform deployments by centralizing the orchestration of disparate internal tooling into a cohesive, automated framework.
