# Project Reporting Bot

This bot aggregates project statistics (GitHub, OpenAI, Anthropic, Gemini) and publishes reports to messaging platforms like Slack and Microsoft Teams.

## Configuration

The bot uses environment variables (loaded via `chat/.env`) for configuration.

### Sample `.env`

```env
# --- General Config ---
PROJECT_ID=E2G-Harness
REPO=mapengbo0314/e2g

# --- Slack Config ---
SLACK_BOT_TOKEN=xoxb-...
SLACK_CHANNEL_ID=C12345678

# --- Teams Config ---
TEAMS_URL=https://prod-XX.westus.logic.azure.com:443/workflows/...

# --- Data Fetchers (API Keys) ---
GITHUB_TOKEN=ghp_...
OPENAI_API_KEY=sk-...
ANTHROPIC_API_KEY=sk-ant-...
GEMINI_API_KEY=AIzaSy...

# --- Cloud Infra ---
AWS_ACCESS_KEY_ID=AKIA...
AWS_SECRET_ACCESS_KEY=...
GCP_SERVICE_ACCOUNT_JSON={"type": "service_account", ...}
```

## Setup Instructions

### Slack Setup
1. **Create App:** Go to [api.slack.com/apps](https://api.slack.com/apps) and click **Create New App** -> **From Scratch**.
2. **Permissions:** In the sidebar, click **OAuth & Permissions**. Scroll to **Bot Token Scopes** and add `chat:write`.
3. **Install:** Scroll up and click **Install to Workspace**. Copy the **Bot User OAuth Token** (`xoxb-...`) to `SLACK_BOT_TOKEN`.
4. **Channel ID:** In Slack, click your channel name at the top. Scroll to the bottom of the **About** tab to find the **Channel ID**. Copy to `SLACK_CHANNEL_ID`.
5. **Invite Bot:** In your Slack channel, type `/invite @YourAppName`.

### GitHub Setup
1. **Token Generation:** Go to [GitHub -> Settings -> Developer Settings -> Personal Access Tokens -> Tokens (classic)](https://github.com/settings/tokens).
2. **Scopes:** Click **Generate new token (classic)**. Select the **`repo`** scope (this covers PRs, issues, and releases).
3. **Save:** Copy the generated `ghp_...` token to `GITHUB_TOKEN`.
4. **Repository:** Ensure the `REPO` env var is in `owner/repo-name` format (e.g., `google/guava`).

### Microsoft Teams Setup
1. **Power Automate:** Go to [make.powerautomate.com](https://make.powerautomate.com).
2. **Create Workflow:** Create a new **Instant cloud flow** triggered by **When an HTTP request is received**.
3. **JSON Schema:** Use a simple schema or leave it blank (the bot sends a standard JSON payload).
4. **Action:** Add a **Post card in a chat or channel** (Teams) action.
5. **URL:** Copy the **HTTP POST URL** generated by the trigger to `TEAMS_URL`.

### AI Usage Setup
- **OpenAI:** Get your key at [platform.openai.com/api-keys](https://platform.openai.com/api-keys).
- **Anthropic:** Get your key at [console.anthropic.com/settings/keys](https://console.anthropic.com/settings/keys).
- **Gemini:** Get your key at [aistudio.google.com/app/apikey](https://aistudio.google.com/app/apikey).

### Cloud Infrastructure Setup
- **AWS:** Create an IAM user with `arn:aws:iam::aws:policy/JobFunction/Billing` or `ce:GetCostAndUsage` permissions. Generate an Access Key/Secret.
- **GCP:** Create a Service Account in the [GCP Console](https://console.cloud.google.com/iam-admin/serviceaccounts), grant it Billing viewer access, and download the JSON key. Paste the *entire* JSON content into `GCP_SERVICE_ACCOUNT_JSON`.

## Running the Bot

Always run from the **project root** using the `-m` flag to ensure imports resolve correctly:

```bash
# Activate your venv first
source .venv/bin/activate

# Install dependencies if you haven't
pip install -r chat/requirements.txt

# Run the bot
python -m chat.main
```
