# Design-as-Code: Phase 1 (Discovery)

**Goal:** Understand the problem and ensure it fits the existing Ubiquitous Language.

## Instructions
1. **Prerequisite:** This skill should ONLY be activated *after* `brainstorming` has begun and the `@architect` subagent has been used to map the initial codebase context.
2. **Domain Alignment:** You MUST invoke the `grill-with-docs` skill. Validate the user's initial idea against `CONTEXT.md`, `ARCHITECTURE.md`, or existing ADRs.
3. **Synthesis:** Write a high-level summary of your understanding of how the proposed change fits into the existing domain model.
4. **Transition:** Once verified by the user, proceed to `dac-2-proposal`.
