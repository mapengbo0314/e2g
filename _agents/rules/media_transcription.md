# Media Transcription Rules

## Purpose

Govern how videos, screenshots, and walkthrough recordings are converted into
project artifacts.

## Rules

1. Preserve filenames, directory names, and timestamps when they are visible.
2. Separate raw evidence from interpreted code or design suggestions.
3. Prefer notes when the source is ambiguous.
4. Route extracted content through the planner before large multi-file edits.
5. Track migration-relevant findings when Python behavior suggests a future
   Kotlin or Java model.
