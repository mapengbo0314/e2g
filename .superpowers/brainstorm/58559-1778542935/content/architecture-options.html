Slack Integration Architecture Options

Where should this integration live in the Superpowers Harness?

A

Standalone Extension (Recommended)

A separate module that queries the git repo, `artifacts/` folder, and cloud APIs. Keeps core harness light.

B

Native Harness Agent

A new "reporter" agent (`@metrics-reporter`) that compiles this data as part of the standard lifecycle.

C

CLI Plugin / Command

An extension to the existing CLI (e.g., `harness-wf report --slack`) run via cron or CI/CD.
