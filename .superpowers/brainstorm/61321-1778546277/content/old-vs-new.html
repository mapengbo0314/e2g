Evolution: From Basic CLI to The River

Comparing the current Harness state to the target architecture

Old/Current Architecture (Siloed)
Agentic Harness
Executes tasks locally.

Logs printed to terminal only.
Metrics buried in `artifacts/`.
Token costs calculated via scripts.
Zero external visibility.

If you want a dashboard, you have to manually parse `artifacts/qa_report.md` and git logs, then build a custom script to push it.
New Architecture (The River + Rollup)
Agentic Harness
Executes tasks locally.

↓ Emits standardized JSON events
~/.gemini/logs/telemetry.jsonl

Integration Daemon (New Component)
Reads `telemetry.jsonl` and branches:
  • Branch A (River): Streams raw events to `#river` channel for team learning.
  • Branch B (Rollup): Aggregates data + AWS/GCP APIs for weekly executive dashboards.

The "Old Architecture" is what we have right now: a great local tool that tells nobody what it's doing. The "New Architecture" introduces the Decoupled Telemetry Layer and Integration Daemon to support the Shopify River style while keeping the dashboard option alive.
