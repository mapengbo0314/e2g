# Verification & Remediation Report: Discovery & Minting Alignment

## 1. Summary of Work
The Agentic Harness was verified against Gemini CLI/Claude Code documentation and the "Phased Onboarding" design. Critical bugs were identified by the `@verifier` and `@adversary` agents, which have now been remediated.

## 2. Remediation Details

### 2.1 SME Synthesis Fix
- **Issue**: Regex mismatch prevented capturing invariants from `ONBOARDING_DOMAIN.md`.
- **Fix**: Updated `synthesize_domain_sme_agent` to use flexible regex matching the bold markdown generated by the discovery engine.
- **Verification**: `tests/test_minting_engine.py::test_synthesize_domain_sme_agent` now passes with the new format.

### 2.2 Claude Tool Mapping
- **Issue**: Specialized agents on Claude were missing search and indexer tools.
- **Fix**: Updated `mint_workspace` to include `Glob` and 6 `mcp_indxr_*` tools in the Claude-specific tool list.
- **Verification**: Verified via source code review.

### 2.3 Robust Rule Patching
- **Issue**: Brittle syntax and poor fallback placement for the SME gateway rule.
- **Fix**: Updated `patch_orchestrator_rules` to use platform-aware syntax (e.g., `Task tool:` vs `@`) and improved insertion logic using `</orchestration_hierarchy>` or DDD section anchors.
- **Verification**: Logic updated in `cli.py` to pass the correct syntax.

### 2.4 Tech Stack Detection
- **Issue**: Design requirement for tech stack detection was missing.
- **Fix**: Implemented `detect_tech_stack` in `discovery_engine.py` (supporting Node, Rust, Python, Go) and integrated it into the `ONBOARDING_DOMAIN.md` template.
- **Verification**: Verified via `test_discovery_engine.py`.

### 2.5 Reliability & Robustness
- **Graceful Failures**: `fetch_remote_skill` no longer exits the process on network failure.
- **Placeholder Validation**: The CLI now warns the user if they attempt to proceed with `[USER INPUT REQUIRED]` placeholders in their onboarding doc.
- **Shell Safety**: Switched to `shlex.quote` for project path escaping in setup scripts.
- **Context Depth**: Added `--detailed` flag to `cli.py` to allow reading all wiki files during discovery.

## 3. Final Verdict
**PASS** - The harness implementation is now structurally sound, aligned with external standards, and robust against common onboarding errors.
